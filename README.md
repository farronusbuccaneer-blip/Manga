# 4コマ漫画吹き出しエディタ (Manga Speech Bubble Editor)

4コマ漫画（3:4比率）をアップロードし、カスタム形式のマークダウンテキストから吹き出し（セリフ）をパースし、各コマへ自動分散配置して編集・保存できる静的Webアプリケーションです。

## 主な機能

1. **マークダウンからのテキスト自動配置**
   - `<section1>`〜`<section4>` および `<row1>`〜`<rowN>` 形式のタグをパースし、4コマに自動で分散してセリフを初期配置します。
   - セリフ内の `/` 記号を自動的に改行に変換します。

2. **3:4 から 4:5 へのアスペクト比自動変換**
   - 3:4（縦長）の漫画画像をアップロードすると、エディタ上および保存時に自動で左右に白い余白を追加し、SNS等で共有しやすい **4:5比率** に変換します。

3. **多様な吹き出しテンプレート**
   - **楕円**: 通常のセリフ用
   - **角丸長方形**: ナレーションやシステムテキスト用
   - **もこもこ（思考）**: キャラクターのモノローグ・心の中の声用（しっぽは3連の泡になります）
   - **ギザギザ（叫び）**: 大声や叫び声用（しっぽを引っ張ると、ギザギザのトゲの1つが伸びて対象を指します）
   - **テキストのみ**: 擬音や背景の文字用（透明背景、枠線なし）

4. **直感的な操作・高度な編集**
   - **ドラッグ＆ドロップ**: 吹き出し本体を掴んで自由に配置可能。
   - **サイズ変更**: 8方向のハンドルをドラッグして、幅や高さを自由に調整可能。
   - **しっぽの個別ドラッグ**: 吹き出しのしっぽ（指示線）の先端だけをドラッグして、喋っているキャラクターの口元を正確に指すことができます。
   - **文字・枠のカスタマイズ**: 文字サイズ（数値＋スライダー指定）、フォント選択、文字揃え、文字色、背景色、枠線色、枠線の太さ、最前面/最背面移動、複製、削除などを個別に調整できます。

5. **完全レスポンシブ & PC・スマホ対応**
   - `PointerEvents` APIを採用しているため、PCのシームレスなマウス操作と、スマホ・タブレットのタッチ操作（ピンチやスクロールと競合しない快適なドラッグ）の両方に完全対応しています。
   - モバイル画面ではエディタとプレビューが上下に折りたたまれるレスポンシブレイアウトになります。

6. **静的HTML + 高解像度保存**
   - バックエンドサーバー不要、すべてブラウザのクライアントサイドのみで完結（HTML/CSS/JS）。
   - Canvas APIを使用して、アップロードされた元の画像の解像度を維持したまま、2倍〜3倍の超高画質で吹き出しを重ね合わせてPNG保存できます。

## 使い方

1. `index.html` をブラウザで開きます。
2. 画面中央のキャンバスに、お手持ちの4コマ漫画画像（3:4比率）をドラッグ＆ドロップ（またはファイル選択）してアップロードします。
3. サイドバーのテキストエリアに、指定形式のマークダウンを入力します。
   ```html
   <section1>
     <row1>We need to check the... what was it... / 私たちは確認する必要があります、その…何だっけ…</row1>
     <row2>Oh no! The word completely slipped my mind! / やばい！単語が完全に頭から消えた！</row2>
   </section1>
   ```
4. **「テキストを解析して配置」** ボタンを押します。自動的にセリフがキャンバス上に生成されます。
5. キャンバス上の吹き出しをクリックして選択すると、右側のサイドバーが **「吹き出し編集」** タブに切り替わります。
6. 位置、サイズ、吹き出しの種類、しっぽの位置などを自由にカスタマイズします。
7. **「4:5画像をダウンロード」** ボタンをクリックして、編集後の画像を保存します。

### ショートカットキー
- 吹き出しを選択した状態で `Delete` または `Backspace` キーを押すと削除できます。
- `Esc` キーを押すと選択を解除できます。

## GitHub Pages へのデプロイ方法

このアプリは静的HTMLのみで構成されているため、GitHubにリポジトリを作成してアップロードするだけで即座に公開できます。

1. GitHubで新規リポジトリを作成します。
2. 本プロジェクトのファイル群（`index.html`、`styles.css`、`app.js`）をコミット＆プッシュします。
3. リポジトリの **Settings** ＞ **Pages** に移動します。
4. **Build and deployment** ＞ **Source** で `Deploy from a branch` を選択します。
5. **Branch** に `main`（または `master`）、フォルダに `/ (root)` を指定して **Save** をクリックします。
6. 数分後、`https://<ユーザー名>.github.io/<リポジトリ名>/` にてアプリが自動公開されます。
